The JS Paywall hides content on any Ghost post or page so that only members who have purchased access can read it. It works without Stripe in Ghost — your payment provider handles the purchase, and PayGlue grants Ghost membership automatically after the transaction.

## How it works

A small script in your Ghost site header checks whether the logged-in member has active access. If not, a paywall overlay appears over the locked content. The overlay disappears as soon as the member logs in with an account that PayGlue has already granted access to after a purchase.

## What you need

Before embedding the Paywall, make sure the following are in place:

- Ghost is connected in PayGlue (Admin API key added)
- A payment provider is connected
- At least one product and mapping are set up in PayGlue
- The PayGlue header script is installed in Ghost (Step 1 below)

The overlay will display as soon as the header script and embed are in place. Members can only unlock the content once the full payment flow is working — provider connected, product mapped, webhook active.

<Warning>
  The JS Paywall is client-side only. A technically capable visitor can use browser DevTools to read the hidden content without paying. It is suitable for blog posts, newsletters, and community content — not for sensitive data like credentials or medical records.
</Warning>

## Setup

<Steps>
  <Step title="Install the header script in Ghost">
    In your PayGlue dashboard, go to **Installation** and copy the header script.

    PLACEHOLDER IMAGE - PayGlue Installation page showing the header script code block (`<script src="https://api.payglue.io/paywall.js" data-org="...">`) with the Copy button highlighted

    Open Ghost Admin and go to **Settings → Code Injection → Site Header**. Paste the script and click **Save**.

    PLACEHOLDER IMAGE - Ghost Admin Settings → Code Injection screen. The Site Header text field contains the PayGlue script tag. The Save button is visible in the top right.

    The script only needs to be installed once. It runs on every page of your Ghost site automatically.
  </Step>
  <Step title="Create a Paywall config">
    In your PayGlue dashboard, go to **JS Paywall** and click **New paywall**.

    Set the headline, body text, button label, and button URL. The button URL should point to your checkout page so visitors can purchase access directly from the overlay.

    PLACEHOLDER IMAGE - JS Paywall config form in the PayGlue dashboard. Fields visible: Name, Headline, body text, button label, button color picker, and button URL. A live preview of the overlay is shown on the right side.

    Click **Save**. The embed snippet appears below the form.

    PLACEHOLDER IMAGE - Saved Paywall config in edit mode showing the embed snippet (`<div data-payglue-gated ...>`) below the form with a copy button
  </Step>
  <Step title="Embed in your Ghost post">
    Open the Ghost post or page where you want to protect content.

    Add an **HTML card** at the point where you want the paywall to begin. Paste the embed snippet from Step 2 into the card.

    PLACEHOLDER IMAGE - Ghost post editor. An HTML card is inserted mid-article. The PayGlue Paywall embed snippet is pasted inside the card. Content below the card is visually cut off to indicate it will be hidden.

    Everything above the HTML card is visible to all visitors. Everything below the card is hidden behind the paywall overlay.
  </Step>
  <Step title="Test the paywall">
    Open the post in a private or incognito window. The paywall overlay should appear over the content below the embed.

    PLACEHOLDER IMAGE - Live Ghost blog post viewed in an incognito browser. The first paragraphs are visible, then a PayGlue overlay covers the rest with the configured headline, body text, and a buy button.

    Log in with a Ghost member account that has active PayGlue access. The overlay should disappear and the full content should be visible.
  </Step>
</Steps>

## Customizing the overlay

You can change the headline, body text, and button style in the Paywall config at any time. Changes apply immediately to every post that uses that config — no need to update the embed in Ghost.

## Using multiple Paywall configs

You can create multiple configs — for example, one per product or content category. Each config has its own embed snippet, so different posts can show different overlay text or link to different checkout pages.

## Content above the paywall is always public

Only the content placed after the embed snippet is hidden. Use this to show a teaser paragraph or preview image above the embed, so visitors understand what they are buying before they see the overlay.
