---
title: "Member did not receive access"
description: "Steps to diagnose why a member did not get Ghost access after a purchase."
---

## Check the event log first

Go to **Analytics > Webhook Events** in your PayGlue dashboard. Find the event for the purchase and check the status.

## Common causes

| Status | Cause | Fix |
| --- | --- | --- |
| Failed | Ghost Admin API error | Check your Ghost credentials under **Connection > Ghost CMS** |
| Skipped | Event type not mapped | Open the Buy Button, Paywall, or Pricing Table item for this product and configure the Ghost tier |
| No event | Webhook not received | Check the webhook URL in your provider's settings |

## Ghost credentials expired

If your Ghost Admin API key was regenerated or revoked, PayGlue cannot create members. Update your credentials under **Connection > Ghost CMS**.

## Webhook URL mismatch

If you renamed your organization slug, your webhook URLs changed. Update the URL in your provider's webhook settings. The correct URL is shown on each provider's connection page under **Connection**.
